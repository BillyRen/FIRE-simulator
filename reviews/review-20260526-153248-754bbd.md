## High

### 1. MCP simulation tools bypass backend-equivalent numeric validation

- Path: `mcp_server/tools.py`
- Line: `119`
- Severity: high
- Category: Diff
- Description: MCP tools only cap `num_simulations` from above and otherwise bypass the backend Pydantic bounds. Inputs like `num_simulations=0`, negative `initial_portfolio`, negative `annual_withdrawal`, `target_success=2`, or `leverage=100` are accepted or crash later with raw errors.
- Suggested fix: Reuse backend request schemas or add shared MCP validators matching `backend/schemas.py` for all numeric parameters.

### 2. Withdrawal sweep accepts zero, inverted, or unbounded ranges

- Path: `mcp_server/tools.py`
- Line: `183`
- Severity: high
- Category: Diff
- Description: `fire_sweep_withdrawal` does not validate `rate_min`, `rate_max`, or `rate_step`. `rate_step=0` raises `ZeroDivisionError`; tiny steps or huge ranges can create very large arrays or loops.
- Suggested fix: Enforce `rate_step > 0`, bounded `rate_max`, `rate_min <= rate_max`, and a maximum number of sweep points before calling `sweep_withdrawal_rates`.

### 3. Runtime configuration is scattered and silently defaults

- Path: `backend/main.py`, `frontend/src/lib/api.ts`, `simulator/sweep.py`, `simulator/config.py`
- Line: `39`; `1`; `38`; `181`
- Severity: high
- Category: Holistic
- Description: Runtime configuration is spread across backend, frontend, and simulator modules and can silently fall back when production values are missing or malformed. Production can boot with missing or invalid `ALLOWED_ORIGINS`, `NEXT_PUBLIC_API_URL`, `MAX_SWEEP_WORKERS`, or `MEMORY_LIMIT_MB`.
- Suggested fix: Add typed backend and frontend settings modules, validate required production environment variables at startup or build time, and document all supported variables in `.env.example`, README, and agent docs.

### 8. Python import boundaries rely on `sys.path` mutation

- Path: `backend/main.py`, `mcp_server/server.py`, `mcp_server/helpers.py`
- Line: `16`; `16`; `16`
- Severity: medium
- Category: Holistic
- Description: Backend and MCP import boundaries depend on `sys.path` mutation and unqualified imports like `from deps import ...`. The simulator and backend are not packaged via `pyproject.toml`, making execution context fragile.

### 10. Streaming errors can expose internal exception details

- Path: `backend/deps.py`
- Line: `78`; `94`
- Severity: medium
- Category: Holistic
- Description: Streaming error handling catches generic exceptions, logs them, and returns `str(exc)` to clients. Error envelopes also differ between streaming and non-streaming endpoints.
- Suggested fix: Add centralized exception handlers and a shared safe error schema. For generic exceptions, return a stable public message or code while logging details server-side with request context.

### 11. Backend observability is missing for expensive API paths
