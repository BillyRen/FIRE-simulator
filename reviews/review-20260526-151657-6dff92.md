## Medium

### 3. MCP simulation tools lack FastAPI-equivalent input bounds

- Path: `mcp_server/tools.py`
- Line: `88`
- Severity: medium
- Category: Diff
- Description: The new MCP tools expose expensive simulation paths without the numeric bounds enforced by the FastAPI schemas. Inputs such as very large `retirement_years`, tiny `rate_step`, huge `leverage`, or `num_simulations <= 0` can cause memory exhaustion, long CPU loops, or uncaught errors.
- Suggested fix: Add shared MCP validation mirroring `BaseSimulationParams` and sweep-specific limits: `retirement_years` between 1 and 100, bounded `num_simulations`, `leverage` between 1 and 5, positive portfolio and withdrawal values, valid target and guardrail ranges, `rate_step > 0`, and a maximum number of sweep points.

### 4. Guardrail batch denominator note can misrepresent baseline metrics

- Path: `frontend/src/app/guardrail/page.tsx`
- Line: `806`
- Severity: medium
- Category: Diff
- Description: The guardrail batch UI displays one denominator note using only `num_incomplete_failed_g` and `num_excluded_g`, but baseline success and funded metrics are computed with separate baseline eligibility counts. When guardrail and baseline failures differ, the note explains the wrong denominator for baseline metrics.
- Suggested fix: Show separate denominator text for Guardrail and Baseline, or include both G and B failed/excluded counts in the message.

### 5. CMA import writes generated CSVs before validating them

- Path: `scripts/import_horizon_cma.py`
- Line: `288`
- Severity: medium
- Category: Diff
- Description: The import script writes generated CSVs before running validation. If PSD, bounds, shape, or plausibility validation fails, invalid files are already left in `data/cme` or the requested output directory.
- Suggested fix: Run `validate()` before writing files, or write to temporary files and atomically replace target CSVs only after validation succeeds.

### 6. Guardrail censored-aware aggregation tests miss denominator behavior

- Path: `tests/test_backtest_batch.py`
- Line: `547`
- Severity: medium
